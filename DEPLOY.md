# 手机长期访问方案

## 结论

当前 `file://` 链接只属于 Mac 本机文件，Mac 关机后手机无法访问。

要让手机在 Mac 不开机时也能正常访问，需要把这个文件夹发布到一个持续在线的 HTTPS 静态托管服务。

## 推荐方案

### 方案 A：GitHub Pages

适合：免费、稳定、后续自己能维护。

步骤：

1. 新建一个 GitHub 仓库，例如 `bahrain-private-kitchen`。
2. 上传本文件夹中的这些文件：
   - `index.html`
   - `manifest.webmanifest`
   - `icon.svg`
   - `sw.js`
   - `DEPLOY.md`
3. 在仓库 Settings 中打开 Pages。
4. Source 选择主分支。
5. 等待 GitHub 生成一个 `https://...github.io/.../` 链接。
6. 用手机 Safari 打开链接，点分享按钮，选择“添加到主屏幕”。

### 方案 B：Cloudflare Pages / Netlify / Vercel

适合：想要更像正式网站，后续可能绑定域名。

做法：把本文件夹作为静态站点部署，发布目录就是项目根目录，不需要构建命令。

## 手机安装后怎么用

1. 第一次必须联网打开 HTTPS 链接。
2. 打开成功后，添加到手机主屏幕。
3. 页面会缓存核心文件；再次打开时，即使网络不好，也可以查看已缓存的菜单。
4. “换一个”的记录保存在手机浏览器本地。

## 重要限制

- 如果只用静态托管，没有账号登录。
- 每台手机的替换记录互不同步。
- 如果你希望多设备同步、登录、云端保存偏好，需要再加后端数据库和账号系统。
- 本页包含个人饮食偏好，不建议发布到公开仓库；如果介意隐私，使用私有仓库加 Vercel/Netlify，或绑定带访问控制的服务。
